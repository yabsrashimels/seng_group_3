Git Repository – Questions and Answers

1. What command initializes a new Git repository?
   git init

Explanation:
The `git init` command creates a new Git repository in the current directory.
It sets up all the necessary Git files and folders (such as the `.git` directory) so that the project can start tracking changes using Git.

---

2. How do you clone an existing repository from GitHub?
   git clone <repository-url>

Explanation:
This command downloads a complete copy of a remote repository (including all files, branches, and commit history) to your local machine.

---

3. How do you check the remote URL of your repository?
   git remote -v

Explanation:
It shows the remote repositories connected to your project and their URLs for fetch and push operations.

---

4. How do you check the current status of files in your repository?
   git status

Explanation:
This command displays the state of the working directory and staging area, showing modified, staged, or untracked files.

---

5. How do you stage a single file for commit?
   git add <filename>

Explanation:
It adds a specific file to the staging area so it can be included in the next commit.

---

6. How do you stage all modified files at once?
   git add .
   OR
   git add -A

Explanation:
These commands stage all new, modified, and deleted files for commit.

---

7. What command is used to commit staged changes with a message?
   git commit -m "your commit message"

Explanation:
This saves the staged changes permanently in the repository with a descriptive message.

---

8. Which command shows the differences between unstaged changes and the last commit?
   git diff

---

9. How do you see the differences for files that are already staged for commit?
   git diff --cached
   OR
   git diff --staged

---

10. Which command displays the commit history?
    git log

---

11. How do you show commit history in one line per commit?
    git log --oneline

---

12. How do you list all local branches?
    git branch

---

13. How do you create a new branch?
    git branch <branch-name>

---

14. How do you switch to an existing branch?
    git checkout <branch-name>
    OR (recommended in newer Git versions):
    git switch <branch-name>

---

15. How can you create a new branch and switch to it at the same time?
    git checkout -b <branch-name>
    OR
    git switch -c <branch-name>

---

16. How do you merge a branch into your current branch?
    git merge <branch-name>

---

17. What happens if there are conflicting changes during a merge?
    Git pauses the merge and marks conflicting files.
    You must resolve the conflicts manually, then stage and commit the resolved files.

---

18. What command is used to pull changes from the remote repository?
    git pull

---

19. What command is used to push your local commits to the remote repository?
    git push

---

20. What should you do if git pull says “Your local changes would be overwritten by merge”?
    You must commit or stash your local changes before pulling.

To commit:
git add .
git commit -m "save changes"

To stash:
git stash

---

21. What is a Git conflict?
    A Git conflict occurs when Git cannot automatically merge changes because different commits modify the same part of a file.

---

22. When does a conflict occur?
    A conflict occurs during merge, rebase, or cherry-pick when:

* Two commits modify the same lines
* One commit deletes a file while another modifies it

---
